## 使用建议

### 1. 检查已跟踪的文件
如果某些文件已经被Git跟踪，需要先移除：

```bash
# 移除已跟踪的 .kiro 目录
git rm -r --cached .kiro/

# 移除已跟踪的开发文档
git rm --cached *_extract.md
git rm --cached *_notes.md
# ... 其他文件

# 提交更改
git commit -m "Remove development documentation from tracking"
```

### 2. 验证 .gitignore
检查哪些文件会被忽略：

```bash
# 查看所有被忽略的文件
git status --ignored

# 检查特定文件是否被忽略
git check-ignore -v .kiro/specs/module-home-and-web-request/requirements.md
```

### 3. 强制添加被忽略的文件
如果某个文件被忽略但你想提交它：

```bash
# 强制添加
git add -f path/to/file

# 或者在 .gitignore 中添加例外
# 例如：保留 README.md 但忽略其他 .md 文件
*.md
!README.md
```

### 4. 清理未跟踪的文件
删除所有被忽略的文件（谨慎使用）：

```bash
# 预览将被删除的文件
git clean -ndX

# 实际删除
git clean -fdX
```

## 注意事项

1. **本地配置文件**: `local.properties` 和 `gradle.properties` 包含本地路径和配置，不应提交
2. **密钥文件**: 所有 `.jks`, `.keystore` 等文件包含敏感信息，绝对不能提交
3. **开发文档**: `.kiro/` 目录包含AI开发过程文档，不需要提交到代码仓库
4. **临时文件**: 编辑器和IDE生成的临时文件会污染仓库，应该忽略

## 团队协作建议

如果你在团队中工作：

1. **统一 .gitignore**: 确保所有团队成员使用相同的 .gitignore
2. **文档管理**: 正式文档放在 `docs/` 目录，开发笔记放在 `.kiro/` 或个人目录
3. **代码审查**: 提交前检查是否有不应该提交的文件
4. **敏感信息**: 使用环境变量或配置文件管理敏感信息

## 总结

更新后的 `.gitignore` 文件可以：
- ✅ 防止开发文档被提交
- ✅ 忽略临时和备份文件
- ✅ 保护敏感信息
- ✅ 保持仓库整洁
- ✅ 提高团队协作效率

记住：`.gitignore` 只影响未跟踪的文件。如果文件已经被Git跟踪，需要先使用 `git rm --cached` 移除。
